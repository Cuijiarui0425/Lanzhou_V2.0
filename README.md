# Lanzhou_V2.0

# 球盘摩擦试验机测控系统

## 1. 项目简介

本项目是为中科院兰州化学物理研究所开发的**球盘摩擦磨损试验机测控系统**，用于摩擦学实验的自动化控制与数据采集。系统通过上位机（C# / WPF）协调 PLC、PCIe 数据采集卡、RS485 应变传感器和工业相机四大硬件子系统，实现对三轴电机（球盘摩擦副 + 加载轴）的运动控制、1000Hz 高频力学信号采集、实时数据处理与显示、自动拍照存档等功能。

**系统架构：**

```

传感器/执行器  ──►  通信总线  ──►  上位机软件
                                    │
  ┌──────────┐    Modbus TCP    ┌───┴────────────────┐
  │  PLC     │◄──────────────►  │  ModbusPlcClient   │
  │ (三轴伺服)│    端口502       │  (手写协议栈)       │
  └──────────┘                  ├────────────────────┤
  ┌──────────┐    PCIe 总线     │  Pcie1805Buffered  │
  │ PCIe-1805│◄──────────────►  │  DaqService        │
  │ (力传感器)│    驱动API       │  (滑动平均滤波)     │
  └──────────┘                  ├────────────────────┤
  ┌──────────┐    RS485         │  StrainTransmitter │
  │ 应变变送器│◄──────────────►  │  Service           │
  │ (力+温度) │    Modbus RTU    │  (CRC16校验)       │
  └──────────┘                  ├────────────────────┤
  ┌──────────┐    GigE/USB3     │  EbusCameraService │
  │ 工业相机  │◄──────────────►  │  (eBUS SDK)        │
  └──────────┘    Vision        └───┬────────────────┘
                                    │
                              MainViewModel
                            (状态机 + 数据融合)
```

## 2. 项目代码功能总结

### 2.1 PLC 通信模块 (`PLC/`)

| 文件 | 功能 |
|------|------|
| `ModbusPlcClient.cs` | **手写 Modbus TCP 协议栈**：从零实现 MBAP 报头构建、FC01/02/03/04/05/06/16 功能码、Float32 双寄存器编解码（低字在前）、事务 ID 回显校验、超时控制（800ms）、IO 互斥锁串行化。提供 `ReadMvpSnapshotAsync()` 一次性读取三轴状态/速度/力矩/报警码等全部实时数据。 |
| `PlcPointMap.cs` | **PLC 点表注册中心**：从 CSV 加载语义化 I/O 标签映射（Key → Modbus 地址/数据类型/标定系数），支持 Coil/DiscreteInput/HoldingRegister/InputRegister 四大区域。 |
| `PlcErrorCodeCatalog.cs` | **PLC 错误码字典**：从 CSV 加载错误码与中文描述的映射表。 |
| `ServoAlarmCatalog.cs` | **伺服报警码字典**：从 CSV 加载报警码（支持十六进制解析），包含故障名称、原因分析和处理建议。 |

### 2.2 数据采集模块 (`DAQ/`)

| 文件 | 功能 |
|------|------|
| `Pcie1805BufferedDaqService.cs` | **PCIe-1805 高频采集服务**：后台线程以 1000Hz 轮询采集卡，预分配 `double[8][4096]` 环形缓冲区实现 O(1) 滑动平均滤波，支持 `voltage × ScaleFactor + Offset` 电压-物理量标定转换，NaN/Infinity 哨兵值过滤。 |
| `DaqConfig.cs` | 采集配置 DTO：设备号、通道范围、采样率、滤波窗口、标定系数。`Normalize()` 方法做参数合法性校正。 |
| `DaqSample.cs` | 采样数据值对象：时间戳 + 多通道工程值数组，每次返回深拷贝防止跨线程数据竞争。 |

### 2.3 传感器模块 (`Sensors/`)

| 文件 | 功能 |
|------|------|
| `StrainTransmitterService.cs` | **RS485 Modbus RTU 应变变送器驱动**：手写 RTU 帧构建（SlaveID + FC03 + 地址 + 数量 + CRC16）、CRC16/Modbus 校验（多项式 0xA001）、deadline 驱动的串口接收循环、双通道 Int32 解码（牵引力 + 温度）。10Hz 后台轮询。 |

### 2.4 视觉模块 (`Camera/`)

| 文件 | 功能 |
|------|------|
| `EbusCameraService.cs` | **工业相机采集服务**：eBUS SDK 封装，支持 GigE Vision/USB3 Vision 协议，专用抓图线程（`IsBackground`），像素格式自适应（RGB8/BGR8/Bayer 转换），10Hz UI 刷新节流，`Marshal.Copy` 非托管→托管内存拷贝，反射兼容层适配多版本 SDK。 |
| `CameraFrame.cs` | 帧数据值对象：宽/高/步长/像素格式/像素字节数组。 |
| `CameraDeviceDescriptor.cs` | 设备描述符：封装设备发现结果。 |

### 2.5 主控 ViewModel (`MainViewModel.cs`)

核心协调层（约 6000 行），职责包括：

- **有限状态机（FSM）**：`Idle → Connected → Loading → Running → Stopping → Fault / EStop`，包含急停互锁、模式切换前置条件校验、错误恢复流程
- **七种试验模式**：恒速、往复间歇、阶梯变速、加载系统、点动、自动恒载程序、自动变速程序
- **PLC 轮询与边沿检测**：后台异步轮询 PLC 快照，`Interlocked` 标志位合并 UI 派发防止队列堆积，Motion Done/Fail 边沿触发状态跳转
- **传感器反馈回写**：`SemaphoreSlim` 节流门控，以 5Hz 频率将力传感器数据回写至 PLC，防止总线拥塞
- **自动拍照握手协议**：检测 PLC 拍照请求 → 抓取相机帧 → 存储 JPEG → 回写完成标志 → 等待 PLC 清除请求，含重入保护与 2s 重试节流
- **报警聚合**：优先级分层（急停 > 本地异常 > PLC/伺服报警），伺服 DWord 报警码解码，实时状态位综合
- **用户参数持久化**：JSON 序列化 + `DispatcherTimer` 防抖存储

### 2.6 配置文件 (`config/`)

| 文件 | 用途 |
|------|------|
| `pointmap_mvp.csv` | PLC Modbus 点表：定义所有 I/O 标签的地址、类型、标定 |
| `plc_error_code_map.csv` | PLC 自定义错误码查询表 |
| `servo_alarm_map.csv` | 伺服驱动器报警码查询表（含十六进制码、故障原因、处置方法） |

## 3. 主要工作

### 3.1 工业通信协议

| 技术点 | 项目中的体现 |
|--------|-------------|
| **Modbus TCP** | 从零手写 MBAP 报头 + PDU 构建/解析，实现 FC01~FC16 功能码全覆盖；Float32 双寄存器编解码（Big-Endian 字序、Little-Endian 字节序）；事务 ID 回显校验保证请求-响应配对 |
| **Modbus RTU** | 手写 RTU 帧构建，内联 CRC16/Modbus 校验算法（多项式 0xA001、初值 0xFFFF、LSB 先移位） |
| **串口通信** | RS485 半双工通信，波特率/数据位/校验位/停止位配置，deadline 驱动的分段接收处理 TCP 和串口的数据碎片化 |
| **GigE Vision / USB3 Vision** | 工业相机标准协议，Jumbo Frame 包大小协商 |

### 3.2 多线程并发与实时性

| 技术点 | 项目中的体现 |
|--------|-------------|
| **生产者-消费者模型** | DAQ 后台采集线程（生产者）写入环形缓冲区，UI 线程（消费者）读取最新均值，通过 `lock` + 深拷贝解耦 |
| **互斥与同步原语** | `SemaphoreSlim(1,1)` 串行化 Modbus I/O 防止帧交错；`lock` 保护共享数据；`Interlocked` 原子操作合并 UI 派发 |
| **实时性保障** | `Stopwatch` 高精度计时 + `Thread.Sleep` 软节拍控制采样周期；`volatile` 标志位实现无锁线程退出信号 |
| **线程模型设计** | 类比嵌入式的"中断上下文"与"主循环"剥离——硬件采集在后台线程（≈中断 ISR），UI 更新在主线程（≈主循环），通过队列/标志位通信 |

### 3.3 数据采集与信号处理

| 技术点 | 项目中的体现 |
|--------|-------------|
| **高频采样** | 1000Hz PCIe 采集卡轮询，预分配环形缓冲区避免 GC 抖动 |
| **数字滤波** | O(1) 滑动平均滤波器：维护 running sum，新值加入/旧值移除，无需每次遍历窗口 |
| **传感器标定** | `工程值 = 电压 × 标定系数 + 偏移`，对标嵌入式中 ADC 原始值→物理量的线性标定 |
| **异常值处理** | NaN / Infinity 哨兵值过滤，传感器断线时平滑降级 |

### 3.4 有限状态机（FSM）与系统可靠性

| 技术点 | 项目中的体现 |
|--------|-------------|
| **状态机设计** | 定义完整的状态枚举与合法跳转路径（Idle→Connected→Loading→Running→Stopping→Fault/EStop），每次操作前校验前置条件 |
| **急停与互锁** | EStop 状态具有最高优先级，可从任意状态跳入；恢复需显式 Reset 操作 |
| **断线重连** | Modbus 连接异常时重置状态机至安全状态，PLC 轮询循环容错继续 |
| **防错设计** | 模式切换需满足前置条件（如手动摩擦模式要求先完成加载），参数写入后延迟 350~500ms 等待 PLC 扫描周期锁存 |

### 3.5 底层字节操作与内存管理

| 技术点 | 项目中的体现 |
|--------|-------------|
| **字节序处理** | Modbus Float32 双寄存器拼接（低字在前），`BitConverter` 字节级转换，对标嵌入式中大小端转换 |
| **位运算** | 线圈状态位解析、报警码 DWord 解码（高低字拼接 `(high << 16) \| low`）、CRC 移位异或 |
| **非托管内存操作** | `Marshal.Copy` 从相机 DMA 缓冲区拷贝图像数据到托管内存，类比嵌入式的 DMA 传输 + `memcpy` |
| **零拷贝与预分配** | 环形缓冲区在初始化时一次性分配，运行时无动态内存申请，减少 GC 暂停（对标嵌入式禁用动态分配的实践） |

### 3.6 驱动与硬件抽象

| 技术点 | 项目中的体现 |
|--------|-------------|
| **设备驱动封装** | 各硬件子系统（PLC/DAQ/串口传感器/相机）封装为独立 Service 类，统一 `Initialize/Start/Stop` 生命周期接口 |
| **硬件抽象层** | 通过 CSV 配置点表实现 I/O 地址与业务语义的解耦，更换 PLC 型号只需修改配置文件 |
| **多版本兼容** | 相机 SDK 通过反射适配不同 API 版本，Modbus 读取通过 FC01/FC02/FC03/FC04 多路径回退兼容不同 PLC 固件 |

### 3.7 系统集成与工程实践

| 技术点 | 项目中的体现 |
|--------|-------------|
| **多传感器数据融合** | PLC 实时数据（速度/力矩/状态）+ PCIe 力传感器 + RS485 应变/温度 + 相机图像，时间对齐后统一呈现 |
| **闭环反馈控制** | 力传感器采集值经滤波后以 5Hz 回写至 PLC，实现上位机参与的力反馈闭环 |
| **报警分级与诊断** | 三级报警聚合（急停 > 软件异常 > 硬件报警），伺服报警码结构化解码（hex/故障名/原因/处置建议） |
| **参数持久化** | 用户标定参数、试验配置 JSON 序列化存储，防抖写入防止频繁磁盘 I/O |

## 构建与运行

1. 使用 Visual Studio 打开 `Lanzhou_v1.0.sln`
2. 还原 NuGet 依赖并构建解决方案
3. 在界面中配置 PLC IP 地址（默认 `192.168.1.10:502`）
4. 连接 PLC 后选择试验模式，启动数据采集与运动控制
